Introduction

JavaScript is a lightweight, object-oriented, interpreted programming language that utilizes just-in-time compilation with first-class functions. Best known as the scripting language used for web page behavior, it also has non-browser environment uses as well. Exploring this technical documentation page will help you understand what exactly makes JavaScript the unique programming language that it is. Enjoy!

JavaScript vs. Java

Contrary to popular belief, JavaScript is not "interpreted Java". Java is typically utilized in server-side development, whereas Javascript predominently handles the development of client-side scripting for functions that increase a web page's interactivity. With that said, the creation of of Node.js has allowed developers to increasingly use JavaScript to run server-side scripts for various tools and applications. Some other key differences between JavaScript and Java include:


  • While both Java and JavaScript can run on browsers and servers alike, JavaScript is known to run on browsers in order to enhance the user experience, whereas Java is known to be a web and application server workhorse
  • Java is designed to allow developers to write code once and deploy it to any operating system without having to make changes. While this is not a failproof process, JavaScript is more sensitive to differences in execution environments, such as browser compatibility issues, which requires more individualized coding
  • Java is a language that is compiled prior to being run, meaning it will be compiled to bytecode or machine code before being executed. Thus any structural issues with the code are recognized rather quickly, and once the code is running, developers can attach to the virtual machine and debug in real-time. JavaScript uses just-in-time compilation which means code translation occurs just before runtime. Bugs are then found at runtime and debugging capabilities are highly dependent on the execution environment which will vary widely among developers.

Lexical Structure

It is important to understand the basic rules of how to write programs in a language in order to help the program run effectively and efficiently without creating unnecessary bugs that can be difficult to locate. These rules are terms the lexical structure of Javascript and they include the following:


  • JavaScript is a case-sensitive language, which means paying attention to when and how you capitalize words. For example, the keyword function must be typed as "function", not "Function" or "FUNCTION". When two words are combined in the naming of variables and functions, it is best practice to write in camelCase to easily differentiate words
  • Comments provide context to code and are styled in such a way that JavaScript knows to ignore the text as code is executed. A single-line comment is delimited by "//" at the beginning of the line, whereas a multi-line comment's text falls between a pair of "/*" characters: 
    •             
            // this is a comment

            /* this is also
            a comment */
  • Identifiers are used to name variables, functions, constants, properties, classes and to label certain loops. Identifiers must begin with a letter, underscore (_) or dollar sign ($)
  • Keywords are words that cannot be used as identifiers because they already have special meaning and can only be used in certain contexts in JavaScript. A full list of available keywords is outside the scope of this document, however, examples include const, function, for, get, return and while.
  • The Unicode character set (composed of letters, digits and ideographs) is used in JavaScript to create identifiers, strings and to write comments about the code itself. When certain computer software or hardware cannot display or process a full set of Unicode characters, four-digit Unicode escape sequences can be used. These are written with the characters "\u" followed by four hexicdemical digits or by one to six hexidemical digits enclosed within curly braces. For example, the "é" in "touché" can be written in the following two ways:
    •             
            touch\u00e9
            touch\u{E9}
  • Semicolons are at times considered optional in JavaScript, but they can be necessary for separating statements. For example, in the instance of a single line of code that has more than one statement, a semi-colon is needed to provide separation between them: 
                    
                a = 6; b = 8;
    Generally speaking, JavaScript will interpret a line break as a semicolon when it cannot parse the code as a continuation of the first statement. In situations where a new statement begins with parentheses () or square brackets [] JavaScript can get confused and assign it to the first statement. As such, some programmers will proactively use a semicolon at the beginning of the second line to explictly separate the statements: 
    •                 
                let x = y + f
                ;(a+b).toString()
Variables

A variable is a symbolic name assigned to a value and values can take many different forms. These include:


  • numbers (intergers and real numbers)
  • boolean (true/false) values
  • strings of text within double or single quotation marks
  • null ("no value") values
  • undefined values

Variables are declared with keywords var, let or const. In the code example below, variable x is assigned to the string value "hello world" via the let keyword:

                
                let x = "hello world";
Variable Scope
Scope refers to the extent that a defined variable is visible to other source code. An example of a variable with global scope is one that is defined outside of a function and can be used by any function without passing it to the function as a parameter: 

        const name = "kaitycodes";

Local scope, on the other hand, refers to a variable that is defined within a function and can only be used within the function that defines it: 


        function makeFunc() {
        const myName = "kaitycodes";
        }
